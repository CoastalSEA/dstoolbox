dstoolbox examples

The construct illustrated in the introduction of the dstoolbox is illustrated using the calling function dstb_usage.m and the dstb_DemoData class that inherits from the dscollection abstract class.
Usage of the individual classes is provided in the test_dstoolbox function.

Contents

dm = dstb_usage;    %initialise class that manages calls to models and data classes
run_a_model(dm);
load_data(dm);
plotCase(dm);       %plot some results for a selected Case
displayProps(dm);   %display DSproperties for a selected Case

dstb_usage class

A class to illustrate the combined use of data and model classes that use dstable and dsproperties, with a record for each data set held in dscatalouge. An option to run dstb_usage is included in test_dstoolbox

demoData class

A class to load data from a file and store it in a dstable. The class includes a method to define the dsproperties, read the input file format and load the data into a dstable and plot some output.

demoModel class

A class to run a simple model (2D diffusion using hard code parameter settings) The class includes a methods run the model and save the results, and plot the model output

test_dstoolbox function

The test_dstoolbox function can be used to experiment with the different options available for the dstable, dsproperties and dsdscatalogue. The comments included in the code explain each function call.

Examples of usage

test_dstoolbox(classname,casenum,options)

classname is one of the toolbox classes 'dscatalogue', 'dsproperties', 'dstable'
casenum is the example test case (see below)
options define the input selection for some of the cases (see below)

dscatalogue
casenum and options input arguments not used
Initialises the class object, add and remove some records, prompt to edit case descriptions, select records to be deleted. Results are displayed in the Command Window.

dsproperties
options input arguments not used
casenum defines the test cases below:
Case 1 - create and display blank dsproperties.
Case 2 - set properties and display.
Case 3 - call and set properties individually using UI.
Case 4 - create using a struct and delete contents.
Case 5 - alternative syntax to set all using UI.
Case 6 - assign a struct array.
Case 7 - assign a struct of cell arrays.
Case 8 - set and test loading incomplete struct (fails).
Case 9 - removal of variables and dimensions.
Case 10 - addition of variables and dimensions.
Case 11 - change order of variables and dimensions
Case 12 - input of struct with empty fields

dstable
casenum defines the test cases below:
Case 1 - initialise a blank dstable. option not used
Case 2 - create a simple table with no dimensions. options is a single scalar value
Case 3 - create table with 2d+t array. options are a [1x3] vector
Case 4 - update the values in the variable. options are a [1x3] vector
Case 5 - horzcat and vertcat - simple table with no dimensions. options are a single scalar value
Case 6 - horzcat and vertcat - multiple variables with dimensions. options are a [1x3] vector
Case 7 - add variable rownames, dimensions and metadata in one call. options are a [1x3] vector
Case 8 - access data using dimensions. options are a [1x3] vector
Case 9 - convert dstable to tscollection and back. options are a [1x3] vector

options define the data type used to generate row and dimension data. A single value defines the row dimension and a [1x3] vector defines [row,dim1,dim2] data types. The data types are numbered as follows:
1 - datetime
2 - duration
3 - char
4 - string
5 - numeric

dstb_usage
casenum and options input arguments not used
Runs the model and loads data twice to create a catalogue of four data sets. then calls plot and display functions for selected cases.
